# API Simples em Camadas com Swagger – Exemplo Educacional

Ele dá continuidade ao exemplo anterior, mantendo a mesma estrutura de código, porém adicionando um novo conceito: documentação clara, padronizada e interativa.

## Objetivo do Exemplo

Este exemplo tem como objetivo ensinar:
- Como documentar uma API utilizando Swagger (OpenAPI)
- Como tornar endpoints, parâmetros e respostas claros e testáveis via interface gráfica

## Documentação com Swagger

Neste exemplo, a API é documentada utilizando Swagger, permitindo:
- Visualizar todos os endpoints disponíveis
- Entender parâmetros de entrada e saída
- Ver exemplos de requisição e resposta
- Testar a API diretamente pelo navegador

A documentação Swagger atua como uma fonte de verdade da API, sendo extremamente útil para:
- Desenvolvedores frontend
- Integrações com outros sistemas
- Manutenção e evolução do projeto

## Conclusão

Este exemplo tem caráter educacional, unindo dois pilares fundamentais no desenvolvimento de APIs:
- Boa arquitetura
- Boa documentação

Este projeto não tem como objetivo ser uma API completa de produção, mas sim servir como material didático.
